# Schema Way

## Structure

```
schema/
├── 00_baseline.sql      # Checkpoint baseline (migrations 001-080 consolidated)
├── migrations/          # Post-checkpoint migrations (081+)
│   └── archived/        # 001-080, superseded by baseline — never edit
├── migrations-warm/     # Migrations needing a running graph engine
└── scripts/checkpoint.sh  # Regenerate the checkpoint (replay + dump + verify)
```

## Creating Migrations

1. Create `schema/migrations/NNN_descriptive_name.sql` (next number after 080/latest)
2. Use next sequential number
3. Migrations auto-apply on `./operator.sh restart postgres`

`00_baseline.sql` is generated by `checkpoint.sh` — regenerate it, don't hand-edit.

## Query Language

**Apache AGE uses openCypher** (not Neo4j Cypher):
- No `ON CREATE SET` / `ON MATCH SET`
- See ADR-208 for compatibility notes

## Graph Data Model

```cypher
(:Concept)-[:APPEARS]->(:Source)
(:Concept)-[:EVIDENCED_BY]->(:Instance)
(:Instance)-[:FROM_SOURCE]->(:Source)
(:Concept)-[:IMPLIES|SUPPORTS|CONTRADICTS]->(:Concept)
```

## Namespaces (ADR-606)

- `kg_core` - Graph data (concepts, sources)
- `kg_auth` - Authentication (users, roles, grants)
- `kg_api` - API state (jobs, artifacts, query_definitions)
